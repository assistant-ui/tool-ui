An information + action surface for user-generated content: posts, comments, notes, and threaded messages.

Use Social Post when a tool returns content that should look like **a message authored by someone else**, with avatars, metadata, and optional actions.

## When to use Social Post

Good fits:

- Showing social media posts or comments.  
- Displaying activity feed items (alerts, notifications, user actions).  
- Representing external messages (tickets, emails, chat transcripts) inline.

Avoid Social Post for:

- Dense tables of events (Data Table is better).  
- Editable rich text (use your own editor).

## Rendering tool outputs

### Server – fetch a social post

```ts
import { tool } from 'ai';
import { serializableSocialPostSchema } from '@/components/social-post/schema';

export const tools = {
  showTweet: tool({
    description: 'Show a tweet as a social post',
    outputSchema: serializableSocialPostSchema,
    async execute({ id }) {
      const tweet = await fetchTweet(id);
      return {
        id: tweet.id,
        platform: 'x',
        author: {
          name: tweet.author.name,
          handle: `@${tweet.author.username}`,
          avatarUrl: tweet.author.avatarUrl,
        },
        createdAt: tweet.createdAt,
        body: tweet.text,
        meta: { source: 'X', url: tweet.url },
        metrics: {
          likes: tweet.likeCount,
          replies: tweet.replyCount,
          reposts: tweet.retweetCount,
        },
      };
    },
  }),
};
```

### Client – render and act

```tsx
import { SocialPost, parseSerializableSocialPost } from '@/components/social-post';

function renderParts(message) {
  return message.parts.map((part, i) => {
    if (part.type === 'tool-showTweet' && part.state === 'output-available') {
      const post = parseSerializableSocialPost(part.output);
      return (
        <SocialPost
          key={i}
          {...post}
          actions={[
            { id: 'open', label: 'Open' },
            { id: 'reply', label: 'Reply in app' },
          ]}
          onAction={(id, post) => {
            if (id === 'open') window.open(post.meta?.url, '_blank');
            if (id === 'reply') openReplyComposer(post.id);
          }}
        />
      );
    }

    if (part.type === 'text') {
      return <p key={i}>{part.text}</p>;
    }

    return null;
  });
}
```



## Props

Social Post is driven by a serializable `SerializableSocialPost` schema.

### Content

Social Post is intentionally lightweight. It makes external content feel “native” to your chat stream, then hands off all meaningful actions to your app.
