## When to Use Data Table

- **Good for:** Search results, lists, dashboards, logs with sorting and actions
- **Not for:** Single-record details, complex multi-step forms

## Key Features

- **Responsive**: Table on wide screens, cards on mobile
- **Sorting**: Tri-state: none → asc → desc
- **Formatting**: Currency, dates, links, badges, etc. via declarative config
- **Actions**: Row-level buttons with optional confirmation
- **Accessible**: Screen reader support, keyboard navigation

## Usage

Define a tool that returns table data, then render with the DataTable component.

```tsx
// Backend tool
import { tool } from "ai";
import { serializableDataTableSchema } from "@tool-ui/data-table";

const searchProducts = tool({
  description: "Search products and return results in a table",
  outputSchema: serializableDataTableSchema,
  async execute({ query }) {
    const results = await db.products.search(query);
    return {
      columns: [
        { key: "name", label: "Product", priority: "primary" },
        {
          key: "price",
          label: "Price",
          format: { kind: "currency", currency: "USD" },
        },
        { key: "stock", label: "Stock", align: "right" },
      ],
      data: results.map((p) => ({
        id: p.id,
        name: p.name,
        price: p.price,
        stock: p.stock,
      })),
    };
  },
});

// Frontend with assistant-ui
import { makeAssistantToolUI } from "@assistant-ui/react";
import { DataTable, parseSerializableDataTable } from "@tool-ui/data-table";

export const SearchProductsUI = makeAssistantToolUI({
  toolName: "searchProducts",
  render: ({ result }) => {
    const props = parseSerializableDataTable(result);
    return <DataTable rowIdKey="id" {...props} />;
  },
});
```
