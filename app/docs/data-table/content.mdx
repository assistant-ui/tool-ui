## When to use Data Table

Use Data Table when a tool returns **rows of structured objects** and the user needs to:

- Scan and compare items (search results, issues, accounts, logs).  
- Sort and filter to find what matters.  
- Take row-level actions (open, approve, escalate, and so on).

Avoid Data Table for:

- Single-record detail views (a Media Card or plain text is often better).  
- Complex, multi-step forms (use a dedicated control surface instead).

          name: p.name,
          price: p.price,
          stock: p.stock,
        })),
      };
    },
  }),
};

import { streamText, tool } from 'ai';
import { openai } from '@ai-sdk/openai';
import { serializableDataTableSchema } from '@/components/data-table/schema';

export async function POST(req: Request) {
  const { messages } = await req.json();

  const result = streamText({
    model: openai('gpt-4o'),
    messages,
    tools: {
      searchProducts: tool({
        description: 'Search products and return results in a table',
        outputSchema: serializableDataTableSchema,
        async execute({ query }) {
          const results = await db.products.search(query);
          return {
            columns: [
              { key: 'name', label: 'Product', priority: 'primary' },
              { key: 'price', label: 'Price', format: { kind: 'currency', currency: 'USD' } },
              { key: 'stock', label: 'Stock', align: 'right' },
            ],
            data: results.map((p) => ({
              id: p.id,
              name: p.name,
              price: p.price,
              stock: p.stock,
            })),
          };
        },
      }),
    },
  });

  return result.toDataStreamResponse();
}

// Client: render in chat (app/page.tsx)
import { useChat } from '@ai-sdk/react';
import { DataTable, parseSerializableDataTable } from '@/components/data-table';

export default function Chat() {
  const { messages } = useChat({ api: '/api/chat' });

  return (
    <>
      {messages.map((m) => (
        <div key={m.id}>
          {m.toolInvocations?.map((tool) => {
            if (tool.toolName === 'searchProducts' && tool.state === 'result') {
              const props = parseSerializableDataTable(tool.result);
              return <DataTable key={tool.toolCallId} rowIdKey="id" {...props} />;
            }
            return null;
          })}
        </div>
      ))}
    </>
  );
}

## Features

### Sorting

Tri-state sorting: none → asc → desc (click again to reset). Use controlled mode to sync with filters or URL state.

```tsx
// Uncontrolled: click header to cycle none → asc → desc → none
<DataTable columns={columns} data={data} />

// Controlled: manage sort in parent
const [sort, setSort] = useState<{ by?: keyof Row; direction?: 'asc' | 'desc' }>({});
<DataTable columns={columns} data={data} sort={sort} onSortChange={setSort} />
```

### Responsive layout

Container queries switch between a classic table and mobile-friendly cards. Column priority decides what stays visible in the card header.

```tsx
// Auto (default): table on wide screens, cards on narrow
<DataTable columns={columns} data={data} />

// Force a particular layout
<DataTable layout="table" columns={columns} data={data} />
<DataTable layout="cards" columns={columns} data={data} />

// Mobile priorities
const columns = [
  { key: 'name', label: 'Name', priority: 'primary' },
  { key: 'price', label: 'Price', priority: 'primary' },
  { key: 'category', label: 'Category', priority: 'secondary' },
  { key: 'sku', label: 'SKU', priority: 'tertiary' },
];
```

### Declarative formatting

Keep row values primitive; describe presentation in `columns[i].format`. This keeps UI consistent and tool outputs JSON-serializable.

```tsx
{ key: 'price', label: 'Price', format: { kind: 'currency', currency: 'USD' } }
{ key: 'homepage', label: 'Homepage', format: { kind: 'link', external: true } }
{ key: 'status', label: 'Status', format: { kind: 'badge', colorMap: { open: 'info', closed: 'success' } } }
```

### Actions and receipts
- Add actions to show row-level buttons or menus.  
- Handle clicks in `onAction` and optionally gate them via `onBeforeAction`.  
- Use [Decision Prompt](/docs/decision-prompt) for higher-risk, multi-row approvals and receipts.

```tsx
<DataTable
  rowIdKey="id"
  actions={[
    { id: 'open', label: 'Open' },
    { id: 'archive', label: 'Archive', tone: 'destructive' },
  ]}
  onBeforeAction={async ({ id, row }) => {
    if (id === 'archive') {
      return confirm(`Archive "${row.name}"?`);
    }
    return true;
  }}
  onAction={(id, row) => {
    // Your app logic here
  }}
/>
```

### Accessibility
- Header buttons are focusable; `aria-sort` reflects state for screen readers.  
- A live region announces sort changes; rows and cells read with labels.  
- The table toggles `aria-busy` when loading so assistive tech knows data is pending.  
- Mobile cards preserve relationships between labels and values.



## Props

Data Table accepts serializable rows where each row is a record of JSON primitives (string, number, boolean, null) or arrays of primitives. Keep objects and Dates out of row values; use `format` for presentation.
